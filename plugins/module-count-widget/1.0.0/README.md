# module-count-widget

VYRA plugin that reads the number of installed modules from **v2_modulemanager** and streams the live count into a **home-widget** slot on the **v2_dashboard**.

Primary purpose: validate the cross-scope plugin structure where the backend runs inside `v2_modulemanager` and the UI is rendered inside `v2_dashboard`.

---

## File layout

```
plugins/module-count-widget/1.0.0/
├── manifest.yaml   ← plugin definition, scope, slots
├── schema.json     ← JSON Schema for config_overlay validation
├── icon.svg        ← plugin icon shown in the marketplace UI
├── logic.wat       ← WASM source (readable)
├── logic.wasm      ← compiled WASM binary (generated by wat2wasm)
└── ui/
    ├── index.js    ← Vue 3 component (ES module, no build step)
    └── style.css   ← scoped CSS (.vyra-module-count-widget prefix)
```

---

## Scope topology

| Layer    | Scope type | Target              | Meaning                                      |
|----------|-----------|---------------------|----------------------------------------------|
| Backend  | `MODULE`  | `v2_modulemanager`  | WASM runtime loaded inside the manager       |
| Frontend | `MODULE`  | `v2_dashboard`      | Vue component injected into the `home-widget` slot |

This is intentionally different from the `counter-widget` (which uses `v2_dashboard` for both layers) and tests whether cross-scope plugin routing works correctly.

---

## Data flow

```
v2_dashboard UI
  └─ ModuleCountWidget (setInterval, every 5 s)
       └─ GET /v2_modulemanager/api/modules/instances
            └─ ModulesResponse { total_modules, total_instances, modules: {...} }
```

No server-side push is required — the component polls the REST endpoint on a
configurable interval and updates its reactive state on each successful response.

| `label`              | `string`  | `Installed Modules` | Widget header text                        |

## WASM backend

The WASM module (`logic.wasm`) is a minimal stateful cache shim.  It does **not** make any network calls — those happen exclusively in the Vue frontend.  The WASM exports (`get_count`, `set_count`, `get_instances`, `set_instances`) are available for the PluginRuntime to query or update the cached state if a Python bridge later needs it.

Re-compile from source:

```bash
wat2wasm logic.wat -o logic.wasm
```

## Installation

1. Ensure the plugin appears in `local_repository/index.json` (already done).
2. In the v2_dashboard plugin manager, open **local-repository** and install **Module Count v1.0.0**.
3. The home-widget slot on the dashboard start page will show the live module count.

---

## Validation checklist

After installing the plugin, verify:

- [ ] Plugin appears in the installed plugin list under `v2_modulemanager`
- [ ] The `home-widget` card renders on the v2_dashboard start page
- [ ] The module count matches the output of `GET /v2_modulemanager/api/modules/instances`
- [ ] The count refreshes automatically (default: every 5 seconds)
- [ ] The manual refresh button (↻) triggers an immediate fetch
- [ ] The `OFFLINE` badge appears when the modulemanager API is unreachable
- [ ] Uninstalling the plugin removes the card from the dashboard
